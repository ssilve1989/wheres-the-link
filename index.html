Helping others help you. Because "My PR is failing CI" isn't helpful without a link.

🚫 Stop doing this

🚫 "My build is failing."

🚫 "Why is my API throwing an error?"

🚫 "This code snippet isn't working."

✅ Start doing this

✅ "My build is failing. Here's the link to the logs: [insert link]."

✅ "My API is throwing a 500 error. Here's my endpoint, the payload, and the server logs: [link]."

✅ "This code snippet isn't working, and here’s the full context of my implementation: [link]."

💡 Why it matters

You might think you're saving time by being brief, but you're actually creating more work for everyone (including yourself). A good link or detailed context is like giving a flashlight in a dark room: it helps everyone see what’s going on.

👩‍💻 Simple Rule

  • If you're asking for help and there’s a relevant link—include it.
  • No link? Include everything you can: logs, screenshots, the full context.

"Where’s the link??" is the tech equivalent of "Where’s the fire??" Don’t leave your teammates guessing. ✌️
